La partie bonus de ce sujet consiste à ajouter GitLab dans le laboratoire 
réalisé dans la partie 3. Voici les étapes pour réaliser cette partie bonus :

    1- Installer GitLab localement sur votre machine virtuelle.
    Vous pouvez suivre les instructions d'installation officielles de GitLab
    pour cela : https://about.gitlab.com/install/.

    2- Configurez GitLab pour qu'il fonctionne avec votre cluster Kubernetes.
    Vous devrez ajouter votre cluster Kubernetes à GitLab en suivant 
    les étapes décrites dans la documentation de GitLab : 
    https://docs.gitlab.com/ee/user/project/clusters/.

    3- Créez un espace de noms dédié nommé "gitlab" dans votre cluster Kubernetes.
    Vous pouvez le faire en utilisant la commande suivante :
        -> kubectl create namespace gitlab

    4- Assurez-vous que tout ce que vous avez fait dans la partie 3 
    fonctionne également avec votre instance locale de GitLab.
    Vous devrez peut-être ajuster vos fichiers de configuration et vos 
    scripts pour qu'ils fonctionnent avec GitLab au lieu de GitHub.

    5- Placez tous les fichiers nécessaires pour configurer et déployer 
    GitLab dans un dossier nommé "bonus" à la racine de votre référentiel. 
    Cela inclut les fichiers de configuration, les scripts et les autres 
    éléments nécessaires pour que votre cluster fonctionne avec GitLab.

N'oubliez pas que la partie bonus ne sera évaluée que si la partie obligatoire
est parfaite, c'est-à-dire si elle a été réalisée intégralement et fonctionne
sans dysfonctionnements.
Si vous n'avez pas réussi à remplir toutes les exigences obligatoires,
votre partie bonus ne sera pas évaluée.
