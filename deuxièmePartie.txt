Mandatory part

This project will consist of setting up several environments under specific rules.

It is divided into three parts you have to do in the following order:

• Part 1: K3s and Vagrant
• Part 2: K3s and three simple applications
• Part 3: K3d and Argo CD






Dans la deuxième partie du projet, vous devez mettre en place 3 applications
Web simples dans votre instance K3s, en utilisant une seule machine virtuelle
avec la distribution de votre choix (dernière version stable) et K3s en mode
serveur installé. Voici les étapes pour réaliser cette partie:

    1- Installez K3s en mode serveur sur votre machine virtuelle.

    2- Choisissez 3 applications Web simples que vous souhaitez déployer.
    Vous pouvez utiliser des images de conteneurs existantes sur
    Docker Hub ou créer les vôtres.

    3- Créez les fichiers de configuration YAML pour déployer ces applications
    dans K3s. Pour chacune des applications, vous devez définir les objets
    suivants dans Kubernetes:

        * Deployment: pour déployer et mettre à l'échelle l'application.
        * Service: pour exposer l'application sur un réseau interne ou externe.
        * Ingress: pour router les requêtes HTTP/HTTPS vers les services en fonction
        du nom d'hôte utilisé.

    4- Configurez l'Ingress pour router les requêtes en fonction du nom d'hôte
    utilisé. Par exemple, lorsque le client utilise le nom d'hôte app1.com,
    l'application 1 doit être affichée. Lorsque app2.com est utilisé,
    l'application 2 doit être affichée, et l'application 3 doit être sélectionnée
    par défaut dans les autres cas.

    5- Assurez-vous de déployer plusieurs répliques pour l'application 2,
    comme mentionné dans l'énoncé du sujet.

    6- Appliquez les fichiers de configuration YAML à votre cluster K3s
    en utilisant kubectl apply -f <fichier.yaml>.

    7- Vérifiez que les applications sont déployées et accessibles
    en utilisant les noms d'hôte configurés dans l'Ingress.

Une fois que vous avez terminé cette partie, placez tous les fichiers
de configuration et les scripts nécessaires dans le dossier "p2" à la racine
de votre dépôt Git, comme mentionné dans les instructions de soumission.
