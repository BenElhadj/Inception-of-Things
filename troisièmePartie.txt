Mandatory part

This project will consist of setting up several environments under specific rules.

It is divided into three parts you have to do in the following order:

• Part 1: K3s and Vagrant
• Part 2: K3s and three simple applications
• Part 3: K3d and Argo CD






La troisième partie du projet consiste à utiliser K3d et Argo CD pour 
mettre en place une intégration continue.
Voici les étapes pour réaliser cette partie :

    1- Installez K3D sur votre machine virtuelle.
    K3D nécessite Docker, donc assurez-vous de l'installer également,
    ainsi que les autres logiciels nécessaires.
    Vous devrez écrire un script pour installer tous les paquets et outils
    nécessaires lors de votre défense.

    2- Comprenez la différence entre K3S et K3D.

    3- Créez deux namespaces dans votre cluster Kubernetes :

        * Le premier dédié à Argo CD.
        * Le second nommé "dev" qui contiendra une application.

    4- Créez un dépôt public sur Github où vous pousserez vos fichiers de configuration.
    Assurez-vous d'inclure le nom d'utilisateur d'un membre de votre groupe
    dans le nom du dépôt.

    5- Déployez une application avec deux versions différentes 
    (taguées comme v1 et v2). 
    Vous pouvez utiliser l'application prête à l'emploi créée par Wil 
    disponible sur Dockerhub, ou créer et utiliser votre propre application.
    Si vous créez votre propre application,
    mettez-la à disposition via une image Docker publique poussée
    dans un dépôt Dockerhub.

    6- Configurez Argo CD pour surveiller et déployer automatiquement
    les modifications apportées à votre dépôt Github.
    L'application déployée doit être mise à jour automatiquement lorsque
    vous modifiez la version dans votre dépôt Github.

    7- Lors de l'évaluation, vous devrez montrer que vous pouvez changer 
    la version de l'application depuis votre dépôt Github et vérifier que
    l'application a été correctement mise à jour.

En résumé, dans cette partie, vous devrez maîtriser K3d et Argo CD pour
mettre en place une intégration continue.
Vous devrez également travailler avec des dépôts Github et Dockerhub
pour gérer les versions de vos applications.
